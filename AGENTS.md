# AGENTS.md

## 커밋 & 푸시 지침

사용자가 커밋 & 푸시를 요청하면, 반드시 아래 절차를 따른다.

### README.md 세션 기록 규칙

1. **커밋 전에 반드시** 해당 세션에서 수행한 작업 내용을 `README.md`에 추가한다.
2. 기존 세션 번호에 이어서 새 세션 섹션을 작성한다 (예: 세션 7 다음은 세션 8).
3. 각 세션 섹션에는 다음 내용을 포함한다:
   - 세션 제목 (변경 사항을 요약하는 짧은 제목)
   - 경로: `~/workspace/opencode-chat-router`
   - 대화별로 **사용자 요청 요약**과 **응답/변경 내용 요약**
   - 변경된 파일 목록
4. `전체 타임라인 요약` 테이블에도 해당 세션의 항목을 추가한다.
5. README.md 변경은 코드 변경과 별도 커밋으로 분리한다 (docs 커밋).
